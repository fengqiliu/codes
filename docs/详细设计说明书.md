### 2.3 智能调度模块

#### 2.3.1 调度策略

| 策略名称 | 描述 | 适用场景 |
|---------|------|---------|
| RoundRobin | 轮询调度 | 实例性能一致时 |
| WeightedRoundRobin | 加权轮询 | 实例性能不一致时 |
| LeastConnections | 最少连接数 | 长连接场景 |
| ResponseTime | 最快响应时间 | 延迟敏感场景 |
| ConsistentHash | 一致性哈希 | 需要会话保持时 |

#### 2.3.2 调度算法实现

```java
public interface LoadBalancer {
    ServiceInstance select(List<ServiceInstance> instances, Request request);
}

// 加权轮询实现
public class WeightedRoundRobinBalancer implements LoadBalancer {
    private AtomicInteger counter = new AtomicInteger(0);
    
    @Override
    public ServiceInstance select(List<ServiceInstance> instances, Request request) {
        int totalWeight = instances.stream()
            .mapToInt(ServiceInstance::getWeight)
            .sum();
        
        int index = counter.getAndIncrement() % totalWeight;
        int currentWeight = 0;
        
        for (ServiceInstance instance : instances) {
            currentWeight += instance.getWeight();
            if (index < currentWeight) {
                return instance;
            }
        }
        return instances.get(0);
    }
}
```

#### 4.1.2 RBAC权限模型

```
用户 (User)
  └── 角色 (Role)
        └── 权限 (Permission)
              └── 资源 (Resource) + 操作 (Action)

角色定义:
- SUPER_ADMIN: 超级管理员，全部权限
- ADMIN: 管理员，系统管理权限
- MODEL_MANAGER: 模型管理员，模型相关权限
- API_USER: API用户，接口调用权限
- VIEWER: 只读用户，查看权限
```

### 4.2 数据安全

#### 4.2.1 敏感数据处理

| 数据类型 | 处理方式 | 说明 |
|---------|---------|------|
| 患者姓名 | 脱敏显示 | 显示为"张*明" |

```
  - 数据库字段: AES-256-GCM
  - 文件存储: 服务端加密 (SSE)
  - 密钥管理: HashiCorp Vault
```

```java
@Cacheable(cacheNames = "models", key = "#modelId")
public AIModel getModel(String modelId) {
```

```java
// 异步任务处理
@Async("aiTaskExecutor")
public CompletableFuture<InvokeResult> asyncInvoke(InvokeRequest request) {
    // 1. 创建任务记录
    AITask task = createTask(request);
    
    // 2. 发送到消息队列
    rabbitTemplate.convertAndSend("ai.invoke.queue", task);
    
    // 3. 返回任务ID
    return CompletableFuture.completedFuture(
        InvokeResult.pending(task.getTaskId())
    );
}

// 线程池配置
@Bean("aiTaskExecutor")
public Executor aiTaskExecutor() {
    ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
    executor.setCorePoolSize(10);
    executor.setMaxPoolSize(50);
    executor.setQueueCapacity(1000);
    executor.setThreadNamePrefix("ai-task-");
    return executor;
}
```

## 6. 错误码设计

### 6.1 错误码规范

```
错误码格式: XYYZZZ
- X: 错误级别 (1-系统级, 2-业务级, 3-第三方)
- YY: 模块编号 (01-认证, 02-模型, 03-API, 04-调度)
- ZZZ: 具体错误

示例:
- 101001: 系统内部错误
- 201001: 用户名或密码错误
- 202001: 模型不存在
- 203001: API调用频率超限
- 301001: AI服务调用超时
```
